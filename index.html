Task 1 - Terraform Infrastructure Deployment

🎯 Objective

Download, install, and configure Terraform to launch a basic EC2 instance on AWS, demonstrating Infrastructure as Code (IaC) capabilities and cloud resource management.

🔧 Implementation Phases

Phase 1: Setup & Installation

Download Terraform from HashiCorp, extract to system directory, and configure PATH environment variable for global access.

Phase 2: Environment Configuration

Verify installation using command line, set up AWS credentials, and prepare the working directory structure.

Phase 3: Infrastructure Code Development

Create Terraform configuration files with AWS provider settings and EC2 resource definitions including AMI and instance specifications.

Phase 4: Deployment & Testing

Execute Terraform workflow: initialize, validate, plan, apply, and verify successful EC2 instance deployment.

⚠️ Key Challenges

  • Environment Setup: Proper PATH configuration and system variable management
  • AWS Authentication: Secure credential management and access key configuration
  • Resource Configuration: Correct AMI selection and instance type specification
  • Network Dependencies: Understanding VPC, subnet, and security group requirements
  • State Management: Terraform state file handling and potential conflicts

📊 Expected Results

  • Successful Terraform installation and PATH configuration
  • Functional AWS provider authentication
  • Deployed EC2 instance (t2.micro) with Amazon Linux AMI
  • Proper resource tagging and identification
  • Demonstration of infrastructure lifecycle management

🛠️ Technical Specifications

Platform: Windows AMD64
AWS Region: us-east-1
Instance Type: t2.micro (Free Tier)
AMI: ami-05ffe3c48a9991133 (Amazon Linux)
Key Commands: terraform init, validate, plan, apply, destroy

Task 2 - Secure Ubuntu EC2 Instance with Advanced Networking

🎯 Objective

Launch a secure Ubuntu EC2 instance in AWS with SSH and HTTPS access using the default VPC, latest official Ubuntu AMI, and custom security group configurations for enhanced network security.

🔧 Implementation Phases

Phase 1: AMI Discovery & Configuration

Implement dynamic AMI lookup to fetch the latest Ubuntu 22.04 (Jammy) AMI using data sources and filters for official Canonical images.

Phase 2: Network Infrastructure Setup

Configure default VPC data source and establish security group with proper ingress/egress rules for SSH and HTTPS access.

Phase 3: Security Group Implementation

Create comprehensive security rules for IPv4/IPv6 traffic, SSH access (port 22), HTTPS access (port 443), and outbound traffic policies.

Phase 4: EC2 Instance Deployment

Deploy Ubuntu instance with key pair authentication, public IP association, and security group attachment for secure remote access.

⚠️ Key Challenges

  • Dynamic AMI Selection: Implementing data sources to fetch latest Ubuntu AMI automatically
  • Security Group Configuration: Properly configuring ingress/egress rules for multiple protocols
  • SSH Key Management: Handling key pair variables and secure authentication setup
  • Network Security: Balancing accessibility with security best practices
  • IPv4/IPv6 Dual Stack: Configuring rules for both IP protocol versions

📊 Expected Results

  • Successfully deployed Ubuntu 22.04 EC2 instance with latest AMI
  • Functional SSH access via port 22 with key pair authentication
  • Secure HTTPS access configuration on port 443
  • Properly configured security group with IPv4/IPv6 support
  • Public IP association for external accessibility

🛠️ Technical Specifications

Operating System: Ubuntu 22.04 LTS (Jammy)
AWS Region: us-east-1
Instance Type: t2.micro (Free Tier)
AMI Owner: 099720109477 (Canonical)
Network: Default VPC with Public IP
Security Ports: SSH (22), HTTPS (443)
Key Features: Dynamic AMI lookup, IPv4/IPv6 support
Authentication: SSH Key Pair (kp1)

Task 3 - Multi-Service AWS Infrastructure & Vercel Showcase

🎯 Objective

Create comprehensive AWS infrastructure across 11 different services using free tier resources, document with screenshots, and develop a professional showcase webpage deployed on Vercel to demonstrate cloud engineering capabilities.

🔧 Implementation Phases

Phase 1: Core Infrastructure Services

Deploy foundational AWS services including EC2 (t2.micro), custom VPC with public subnet, Internet Gateway, and EBS volume attachment.

Phase 2: Advanced Services & Databases

Implement serverless Lambda functions, S3 storage with versioning, RDS MySQL database, and Athena analytics with sample data.

Phase 3: Monitoring & Management

Configure CloudWatch alarms, enable CloudTrail for API monitoring, and deploy infrastructure using CloudFormation templates.

Phase 4: Modern DevOps & Documentation

Utilize AWS CDK for infrastructure as code, create comprehensive documentation with screenshots, and deploy showcase on Vercel.

⚠️ Key Challenges

  • Multi-Service Integration: Coordinating 11 different AWS services with proper dependencies
  • Free Tier Limitations: Managing resource constraints while maintaining functionality
  • Infrastructure as Code: Implementing both CloudFormation and CDK approaches
  • Monitoring Setup: Configuring CloudWatch alarms and CloudTrail effectively
  • Documentation Quality: Creating professional screenshots and clear presentation

📊 Expected Results

  • 11 fully functional AWS services deployed and configured
  • Custom VPC with proper networking and security configurations
  • Serverless Lambda function with Python Hello World implementation
  • Comprehensive monitoring with CloudWatch and CloudTrail
  • Professional Vercel webpage showcasing all implementations

🛠️ AWS Services Implemented

EC2: t2.micro Amazon Linux 2 instance
VPC: Custom VPC with public subnet & IGW
Lambda: Python Hello World function
S3: Private bucket with versioning
EBS: 10GB volume attached to EC2
RDS: db.t2.micro MySQL database
Athena: Database & table from S3 data
CloudWatch: CPU utilization alarm >70%
CloudTrail: API activity monitoring
CloudFormation: Template-based EC2 deployment
CDK: TypeScript/Python S3 provisioning
Vercel: Static site deployment platform

🌐 Deployment Architecture

Infrastructure Layer

Custom VPC with public subnet, Internet Gateway, security groups, and network ACLs for secure communication.

Compute & Storage

EC2 instances with attached EBS volumes, S3 buckets for object storage, and RDS for relational database management.

Serverless & Analytics

Lambda functions for serverless computing and Athena for SQL analytics on S3 data lakes.

Management & Monitoring

CloudWatch for metrics and alarms, CloudTrail for audit logs, and CloudFormation/CDK for infrastructure automation.
